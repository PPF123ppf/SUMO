# 混合交通混合博弈换道决策模型 — 技术白皮书

## 1. 问题建模

### 场景

- 4km 三车道高速公路，中间车道 3000m 处事故封堵
- 事故车道上的车辆必须合流到相邻车道
- 混合交通：CAV（渗透率 p）+ 人类车（1-p）

### 决策本质

本车（CAV）要换道时，目标车道后车会如何反应？这不是 "猜概率"，而是 **Stackelberg 序贯博弈**：

```
本车宣布动作（换/不换） → 后车观测后选最优反应 → 本车知晓后车最优反应后做最终决策
```

## 2. 总体架构

```
混合交通流量（CAV 渗透率 p）
│
├── CAV（p%）
│   ├── 感知层 → 噪声/延迟/丢包 V2X 模型
│   ├── 门控层 → 间距门控 + 硬 TTC 安全门控（突发期 2.0s / 有序期 1.5s）
│   ├── 特征层 → 8维原子特征提取
│   ├── 权重层 → 场景自适应加权（密度/紧迫度 → 安全↑ 速度↓）
│   ├── 博弈层 → 混合博弈求解器
│   │   ├── 后车=CAV → 3种博弈结构按后车类型分发
│   │   └── 后车=人类 → 对抗性概率预测 + 互惠记忆
│   ├── 决策层 → 期望收益增益 > 渗透率自适应阈值 → 执行换道
│   ├── 执行层 → 准备(0.4s)→执行(2.8s)→稳定(0.5s) 三阶段状态机
│   └── 学习层 → 在线 TD 微调（每次换道完成后）
│
└── 人类车（1-p%）
    └── SUMO SL2015 原生换道 + Krauss 跟驰 + 紧急制动兜底
```

## 3. 特征空间（8 维原子特征）

每个 `(本车动作=换道/保持, 后车反应=加速/保持/减速)` 组合提取 8 个独立特征。

**设计原则**：每个特征有独立的物理语义，权重可解释。IRL 学到 "safe=0.3" 可直接解释为 "安全偏好 0.3"，优于复合特征 "eff=0.38" 的模糊含义。

| 维度 | 特征 | 换道时的值 | 不换道时的值 | 物理含义 |
|:--:|------|-----------|-------------|----------|
| 0 | **speed** | 目标车道前车速度 / vmax | 自车速度 / vmax | 速度收益 |
| 1 | **urgency** | 距事故归一化距离 (0~1) | 0 | 紧迫驱动力 |
| 2 | **pressure** | 当前车道头距压力 (0~1) | 0 | 逃离压力 |
| 3 | **safe** | min(前向TTC安全, 后向TTC安全) | 当前车道前向TTC安全 | TTC安全评分 |
| 4 | **coop** | 后车协同(0.18) + 后车制动加分(0.12) | 固定 0.05 | 协同奖励 |
| 5 | **social** | 本车切入后，后车TTC缩减比例 | 0 | 社会冲击 |
| 6 | **density** | 局部车流密度 (0~1) | 0 | 环境压力 |
| 7 | **lc_cost** | 1.0 | 0.0 | 换道固有代价 |

### 关键子函数

**safety_from_gap_ttc(gap, rel_speed)**

```
gap_score = clip((gap - min_safe_gap) / 25.0, 0, 1)      # 间距项
ttc_score = clip((gap / rel_speed) / 4.0, 0, 1)           # TTC项
safe = 0.55 * gap_score + 0.45 * ttc_score                # 加权求和
```

**social 特征计算**

```
fol_old_ttc = (fol_gap + lead_gap) / max(fol_spd - lead_spd, 0.1)      # 换道前：后车→原前车
fol_new_ttc = fol_gap / max(fol_spd - ego_spd, 0.1)                     # 换道后：后车→本车
social = clip((fol_old_ttc - fol_new_ttc) / fol_old_ttc, 0, 1)        # TTC缩减比例
```

## 4. 收益函数

### 4.1 本车收益（8 维特征 × 自适应权重）

```
U_e(action, fa) = w_adapt · f(action, fa) - LC_cost × I(action=换道)
```

**基础权重**：

| phase | speed | urgency | pressure | safe | coop | social | density | lc_cost |
|-------|:-----:|:-------:|:--------:|:----:|:----:|:------:|:-------:|:-------:|
| 突发期 (90-100s) | 0.25 | 0.15 | 0.08 | 0.30 | 0.10 | 0.05 | 0.05 | 1.0 |
| 有序期 (100s+) | 0.30 | 0.10 | 0.06 | 0.25 | 0.12 | 0.05 | 0.05 | 1.0 |

### 4.2 自适应权重（场景动态调整）

```python
safe_weight   += 0.10 × density + 0.05 × urgency   # 堵车/近事故 → 更重视安全
social_weight += 0.05 × density                      # 高密度 → 换道对他人影响更大
speed_weight  -= 0.10 × urgency                      # 紧迫时降速求稳
```

| 场景 | speed | safe | social |
|------|:-----:|:----:|:------:|
| 畅通 + 远事故 | 高 | 中 | 低 |
| 拥堵 + 远事故 | 中 | 高 | 中 |
| 拥堵 + 近事故 | 低 | 最高 | 高 |

### 4.3 后车收益（5 维、3 种类型）

```
U_f(ego_action, follower_action) = w_t · [safety, speed_ratio, comfort_cost, coop_reward, gap_safe]
```

**5 维特征**：

| 特征 | 含义 | 计算方式 |
|------|------|---------|
| safety | 追尾本车的 TTC 安全评分 | `safety_from_gap_ttc(gap_to_ego, rel_speed)` |
| speed_ratio | 后车速度有效性 | `fol_spd / fol_max` |
| comfort_cost | 制动不舒适代价 | `max(0, -delta_v / 4.5)`，仅减速时非零 |
| coop_reward | 协同让行社会奖励 | 本车切入 + 后车减速 = 1.0，否则 0 |
| gap_safe | 间距安全底线 | 当前统一为 1.0 |

**3 种后车类型（异质性）**：

| 类型 | 占比 | 博弈结构 | safety | speed | comfort | coop | 行为特征 |
|------|:--:|---------|--------|-------|---------|------|---------|
| Type 0 自私型 | 20% | **静态 Nash** | 0.45 | 0.35 | 0.20 | 0.00 | 无协同动机，双方同时决策 |
| Type 1 合作型 | 60% | **Stackelberg** | 0.40 | 0.25 | 0.15 | 0.20 | CAV 基本协同，本车领导 |
| Type 2 高度合作型 | 20% | **Nash 合作** | 0.35 | 0.15 | 0.15 | 0.35 | 强协同，共同优化联合收益 |

## 5. 混合博弈求解器

### 5.1 求解器入口

```
solve_hybrid_game(ego_payoff, fol_id, ego_speed, fol_gap, lead_spd, phase)
    → 对每种后车类型 t:
        ├── Type 0 → _solve_static_nash(ego, fol)
        ├── Type 1 → _solve_stackelberg_per_type(ego, fol)
        └── Type 2 → _solve_nash_cooperative(ego, fol)
    → 按 FOLLOWER_TYPE_SHARES [0.20, 0.60, 0.20] 加权
    → 返回 expected[换道], expected[保持]
```

### 5.2 三种博弈结构

**静态 Nash（Type 0）**：双方同时决策，无领导者-跟随者关系

```
初始化 ego_strat = [0.5, 0.5]
循环 4 轮：
    后车对 ego 混合策略求最优反应 → fol_strat = softmax(fol_payoff^T @ ego_strat)
    本车对 fol 混合策略求最优反应 → ego_strat = softmax(ego_payoff @ fol_strat)
返回 ego 两个动作的期望收益
```

**Stackelberg（Type 1）**：本车领导者，后车跟随者

```
对每个本车动作 ea ∈ {换道, 保持}:
    后车观测 ea，softmax 选择反应 → fol_action_probs = softmax(fol_payoff[ea] / 0.15)
    本车取期望收益 → expected[ea] = ego_payoff[ea] @ fol_action_probs
```

**Nash 合作（Type 2）**：最大化联合收益

```
joint = ego_payoff + fol_payoff  # 2×3 联合收益矩阵
对每个本车动作 ea:
    后车 softmax 选联合最优 → joint_probs = softmax(joint[ea] / 0.15)
    本车取该反应下的期望 → expected[ea] = ego_payoff[ea] @ joint_probs
```

### 5.3 混合策略（softmax 替代 argmax）

```python
def _softmax(x, temperature=0.15):
    x_adj = (x - x.max()) / temperature
    return exp(x_adj) / sum(exp(x_adj))
```

- temperature=0.15 → 绝大部分概率集中在最优动作，但保留小概率随机性
- 避免硬 `argmax` 导致的确定性僵化
- 模拟人类/CAV 决策中固有的行为噪声

### 5.4 连续后车反应

后车反应幅度不再固定，而是根据间距/速度动态计算：

```python
# 制动强度：间距越紧，制动越猛
brake_dv = -min(4.5, max(1.5, 6.0 / max(fol_gap, 1.0)))

# 加速强度：空间越大，加速越温和
accel_dv = min(2.6, 0.5 + 2.0 * (fol_gap - COOP_MIN_GAP) / COOP_REQUEST_GAP)

delta_vs = (accel_dv, 0.0, brake_dv)
```

## 6. 低渗透率自适应机制

### 6.1 对抗性偏差

后车是人类时，自动将先验向 "不配合" 方向偏移：

```python
# 人类缺乏合作动机 → 加速堵位概率 +10%，减速让行概率 -10%
base += np.array([0.10, 0.00, -0.10])  # [acc↑, maintain, brake↓]
```

### 6.2 互惠记忆

跟踪人类后车的合作历史，动态调整合作预期：

```
换道执行时，检测后车加速度：
    后车减速 > 0.3 m/s² → cooperation_score += 0.20（合作）
    后车加速 > 0.2 m/s² → cooperation_score -= 0.10（不合作）

下次遇到同一辆车：
    extra_coop = (cooperation_score - 0.5) × 0.20
    base += [-extra_coop, 0, +extra_coop]  # 高分→倾向减速
```

### 6.3 渗透率自适应阈值

阈值随 CAV 渗透率动态调整：

```
min_gain = base_min_gain × (1.0 - 0.6 × (1.0 - CAV_PENETRATION))

渗透率 0%   → 0.40× base → 极度果断（全是人类，犹豫=永远出不去）
渗透率 10%  → 0.46× base → 非常果断
渗透率 50%  → 0.70× base → 适度
渗透率 100% → 1.00× base → 正常（全CAV环境可从长计议）
```

## 7. 决策流程

```
① 反应延迟到期 → 车辆进入决策窗口
② 间距门控：lead_gap ≥ dyn_gap 且 fol_gap ≥ dyn_gap
③ 硬安全门控：min(前TTC, 后TTC) ≥ TTC_hard_min
   突发期: 2.0s | 有序期: 1.5s
④ 计算本车收益矩阵 compute_payoff()
   ├── 调用 get_adaptive_weights() 动态调整权重
   ├── 计算连续后车反应 delta_vs
   └── compute_features() 输出 8 维特征 × 6 种组合
⑤ 博弈求解:
   ├── 后车=CAV → solve_hybrid_game()
   │   ├── Type 0 → 静态 Nash
   │   ├── Type 1 → Stackelberg
   │   └── Type 2 → Nash 合作
   └── 后车=人类 → get_follower_prior()
       ├── 对抗性偏差（人类不配合）
       └── 互惠记忆（历史合作者加分）
⑥ 增益 = expected[换道] - expected[保持]
⑦ 增益 > 渗透率自适应阈值 → 执行换道
   换道前记录当前权重到 _online_learning_memory
⑧ 三阶段执行：
   准备(0.4s) → 安全复查 → 执行(2.8s) → 稳定(0.5s) → 完成
⑨ 换道完成/失败 → online_update_weights() 微调权重
```

## 8. 在线学习

每次换道完成后，根据结果即时微调权重（学习率 0.003）：

| 结果 | 调整 |
|------|------|
| 成功完成 | `safe += 0.0003`, `coop += 0.0003`（轻量强化） |
| 安全审查失败 | `safe += 0.0006`（纠偏） |
| 导致后车急刹 | `safe += 0.0024`, `social -= 0.0024`（惩罚） |

学习率极低，不破坏权重主结构，但持续微调可让模型适应不同交通环境。

## 9. 性能优化

| 优化项 | 方法 | 效果 |
|--------|------|:--:|
| 决策减频 | 每 3 步（0.3s）做一次博弈 | 博弈计算量 -67% |
| 速度缓存 | `cached_max_speed()` 同类型只查一次 TraCI | TraCI 调用减少 |
| Nash 迭代减半 | 静态 Nash 迭代 8→4 轮 | Nash 求解器快 ~40% |

## 11. 创新点总结

| 序号 | 创新点 | 关键实现 | 论文价值 |
|:--:|--------|---------|:--:|
| 1 | **8维原子特征** | 每维独立物理语义，IRL 可解释 | 区别于复合特征的"黑箱" |
| 2 | **混合博弈结构** | 3种后车类型 × 3种博弈结构自适应选择 | 纠正"一种博弈打天下" |
| 3 | **混合策略求解** | softmax 替代 argmax，捕捉决策随机性 | 更真实的行为建模 |
| 4 | **Social Impact** | 量化为后车 TTC 缩减比，纳入收益函数 | CAV 的 "社会性" |
| 5 | **自适应权重** | 密度/紧迫度 → 安全↑ 速度↓ | 场景感知决策 |
| 6 | **渗透率自适应** | 低渗透率降低阈值，使 CAV 果断 | 解决 "冻住问题" |
| 7 | **对抗性偏差+互惠记忆** | 人类不配合建模 + 历史学习 | 混合交通关键机制 |
| 8 | **在线 TD 学习** | 每次换道后即时微调权重 | 持续自适应 |
| 9 | **IRL 数据驱动** | 从真人数据学基础权重，手动权重补事故场景 | 两套权重互补叙事 |
