## 模型①：SUMO Default — "SUMO 原生换道"

| 配置项 | 值 | 说明 |
|:------|:--|:-----|
| laneChangeModel | **SL2015** | 子车道换道模型，支持连续横向位置 |
| lateral-resolution | **0.8m** | 子车道分辨率 → 每条3.2m车道=4个子车道 |
| sigma（随机性） | **0.5** | 模拟人类驾驶随机行为 |
| lcSigma | **0.5** | SL2015 特有的驾驶随机性参数 |
| lcStrategic | **1.0** | 路线规划换道全开 |
| lcCooperative | **1.0** | 协同让行全开 |
| lcSpeedGain | **1.0** | 追求速度全开 |
| lcKeepRight | **1.0** | 靠右行驶全开 |
| lcAssertive | **0.5** | 换道激进程度中等 |

**换道思想**：完全交给 SUMO 的 **SL2015 子车道换道模型**。我们不干预任何换道/跟驰决策。仅保留事故创建+紧急刹停兜底。SL2015 支持连续横向位置控制和子车道行驶，比 LC2013 更真实。

**相当于"开箱即用的 SUMO SL2015 子车道模型表现"**

## 模型②：Rule-Based — "固定阈值规则换道"

| 配置项 | 值 | 说明 |
|:------|:--|:-----|
| sigma | **0.0** | CAV精确执行 |
| lc系参数 | 全部=0 | 关闭SUMO原生，由Python控制 |
| **RULE_TTC_THRESHOLD** | **3.0s（硬编码）** | 只要TTC≥3.0s就换道 |
| 协同让行 | ❌ 无 | |
| 感知模型 | ✅ 有（延迟+噪声+丢包） | 与Game模型共享 |
| 冷却时间 | 1.5s | |

**换道思想**：最简单的规则——检查目标车道前后车的 TTC，都 ≥ 3.0s 就换道，否则放弃。

**代表"保守但感知精确"的基线**

## 模型③：Game (Ours) — "同时博弈换道" ★ 本文核心创新1

| 配置项 | 值 | 说明 |
|:------|:--|:-----|
| sigma | **0.0** | CAV精确执行 |
| **换道方式** | **同时博弈（2×3矩阵）** | 本车×后车同时决策 |
| **Level-k认知层级** | ✅有（Level-0/1/2） | 用认知深度优化后车行为预测 |
| **Stackelberg序贯博弈** | ❌ **关** | |
| **Queue顺序协调** | ❌ **关** | 每车独立决策 |
| 协同让行 | ✅ 有 | 后车减速开Gap |
| 编队换道 | ✅ 有 | PLATOON_HEADWAY=15m一起换 |
| TTC硬阈值 | **突发2.0s/有序1.5s** | |
| 换道成本 | 0.05 | 收益函数中换道扣分 |
| 收益函数权重 | 效率0.38+安全0.42+协同0.20 | 有序期 |
| 反应延迟 | 突发0.20~0.50s/有序0.12~0.35s | CAV级别 |
| V2X范围 | 500m→1200m(广播后) | |

**换道思想**：2×3 同时博弈。本车选{换/不换}，后车选{加速/保持/减速}。用 Level-k 认知层级（Level-0=基础预测/Level-1=知道对方Level-0/Level-2=递归认知）优化后车行为先验概率。期望收益超过自适应阈值就换道。

**这是本文的第一个创新点：Level-k 认知博弈换道（剔除了 Stackelberg 和 Queue 两个附加创新）**

## 模型④：No-V2X — "仅局部感知"（Global_V2X降级）

| 配置项 | Game (Ours) | No-V2X |
|:------|:-----------:|:------:|
| 全局广播范围 | **1200m** | **500m（=局部V2X）** |
| 其他所有参数 | — | 完全一致 |

**核心差异**：仅 1 处不同——全局广播范围从 1200m 被强制缩到 500m。事故发生后有序期的全局广播失效，车辆始终只能感知 500m 内的信息。

**模拟没有RSU（路侧单元）基础设施支持的场景**

---

## 模型⑤：LKSQ (Ours) — "三重创新完全体" ★ 最高性能

| 配置项 | Game (Ours) | LKSQ (Ours) |
|:------|:-----------:|:-----------:|
| **Level-k** | ✅ 有 | ✅ 有 |
| **Stackelberg序贯博弈** | ❌ 关 | ✅ **开** |
| **Queue顺序协调** | ❌ 关 | ✅ **开** |
| 其他参数 | — | 完全一致 |

**换道思想**：Game (Ours) 基础上增加两个机制：

**① Stackelberg 序贯博弈**：本车先"承诺"换道/不换道→后车观察到承诺后选择最优响应→本车在知道后车最优响应的情况下决策。比同时博弈更符合实际（先打灯→后车反应→再换道）。

**② 顺序排队协调**：事故车道上所有车按距障碍物距离排成一队，**每次只有队首那辆车可以发起换道**。后面的车依次等待，防止多车同时抢道。
